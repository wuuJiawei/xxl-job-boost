# Release Notes · 2026-06-10

本次发布可以视为 XXL-JOB Boost 的第一版可用基线，重点不是调度内核重构，而是把新管理后台和配套接口真正落到仓库内，形成可运行、可演示、可继续迭代的增强版本。

## 发布摘要

- 发布日期：2026-06-10
- 发布类型：功能增强基线
- 兼容策略：保留旧控制台，新增新控制台入口

## 本次新增

### 1. 新管理后台 `admin-next`

新增 `xxl-job-admin-ui` 前端工程，采用：

- Vue 3
- TypeScript
- Vite
- Pinia
- Naive UI
- ECharts

新控制台已覆盖核心日常页面：

- 登录
- Dashboard
- 执行器管理
- 任务管理
- GLUE 代码管理
- 调度日志列表与详情
- 用户管理
- 帮助页

### 2. 新后端接口层

在 `xxl-job-admin` 内新增 `/api/admin-next/**` 接口，用于支撑新控制台，包括：

- 会话信息
- Dashboard 摘要
- 执行器与任务元数据
- 任务详情
- GLUE 代码详情与保存
- 日志详情
- 用户管理元数据与详情

### 3. 新旧控制台并存

本次没有删除旧控制台，而是保留双入口：

- 旧控制台：`/xxl-job-admin/`
- 新控制台：`/xxl-job-admin/admin-next/`

这保证了迁移过程可回退、可对照、可分阶段切换。

### 4. 本地开发脚本完善

仓库内已提供本地开发脚本，能够快速拉起：

- MySQL 容器
- 调度中心
- 样例执行器

同时补充了前端开发脚本，方便本地独立调试新控制台。

## 本次调整

### 保留的内容

- `xxl-job-admin` 仍是主后端应用
- `xxl-job-core` 调度核心保持原有演进路径
- `xxl-job-executor-samples` 继续作为联调样例

### 仓库结构调整

- 新增 `xxl-job-admin-ui`
- 保留 `xxl-job-admin-ui-legacy`
- 新控制台静态资源挂载到 `xxl-job-admin` 内部

### CI 配置处理

仓库中的 GitHub Actions workflow 已移除。这样做的原因是：

- 当前阶段优先保证仓库代码与功能交付
- 避免默认推送受 PAT `workflow` 权限影响
- CI 方案后续可在权限和发布流程明确后再重新引入

## 对使用方的影响

### 正向影响

- 可以直接体验更现代的管理后台
- 不需要一次性切换旧控制台
- 新前端与新接口分层更清晰，后续继续增强成本更低

### 兼容性说明

- 现有执行器接入方式不要求立即调整
- 现有数据库初始化方式保持不变
- 默认管理员账号仍为 `admin / 123456`

## 当前已知边界

- 当前重点仍然是后台体验升级，不是调度核心能力重写
- 仓库已具备可用的新控制台，但仍属于持续增强中的版本
- 告警、自动注册、治理与可观测性增强尚未在本次集中交付

## 后续方向

后续版本优先级建议如下：

1. 补齐新控制台剩余交互细节和体验统一
2. 增强告警通道与告警记录
3. 增强执行器注册与元数据管理
4. 增强任务治理、审计与观测能力
